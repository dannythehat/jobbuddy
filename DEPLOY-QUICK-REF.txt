│ STEP 1: FIX PERMISSIONS (5 min)                             │
└─────────────────────────────────────────────────────────────┘

chmod +x scripts/fix-gcp-permissions.sh
./scripts/fix-gcp-permissions.sh

→ Creates service account
→ Grants necessary permissions
→ Generates key file: github-actions-key.json

┌─────────────────────────────────────────────────────────────┐
│ STEP 2: ADD GITHUB SECRET (2 min)                           │
└─────────────────────────────────────────────────────────────┘

1. Copy key contents:
   cat github-actions-key.json

2. Go to:
   https://github.com/dannythehat/jobbuddy/settings/secrets/actions

3. Click "New repository secret"
   Name:  GCP_SA_KEY
   Value: [paste entire JSON]

┌─────────────────────────────────────────────────────────────┐
│ STEP 3: CREATE SECRETS (3 min)                              │
└─────────────────────────────────────────────────────────────┘

./scripts/create-secrets.sh

OR manually:

echo -n "your_password" | gcloud secrets create jobbuddy-db-password --data-file=-
echo -n "your_jwt_secret_32_chars" | gcloud secrets create jobbuddy-jwt-secret --data-file=-
echo -n "sk-your-key" | gcloud secrets create jobbuddy-openai-key --data-file=-
echo -n "your_encryption_key" | gcloud secrets create jobbuddy-encryption-key --data-file=-

┌─────────────────────────────────────────────────────────────┐
│ STEP 4: DEPLOY! (1 command)                                 │
└─────────────────────────────────────────────────────────────┘

git push origin main

→ GitHub Actions triggers automatically
→ Builds & tests code
→ Creates Docker images
→ Deploys to Cloud Run
→ Outputs service URLs
